# Cluster Observability Baseline

Workload observability is generated by ArgoCD ApplicationSet cluster generators
for every cluster labeled:

```yaml
devplane.io/workload: "true"
```

The generated applications are declared in:

- `gitops/applicationsets/workload-observability-charts.yaml`
- `gitops/applicationsets/workload-agent-addons.yaml`

Grafana, Loki, Tempo and Mimir come from:

```text
charts/observability/
```

OpenTelemetry Collector and Vector come from:

```text
charts/agents/
```

Prometheus/kube-prometheus-stack is intentionally not part of the workload
baseline. Metrics should be written to Mimir.

Telemetry flow is OpenTelemetry Collector -> Vector -> Loki/Tempo/Mimir. MinIO
is used as object storage for the observability stack where supported by the
packaged charts.
